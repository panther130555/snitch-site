Privacy Policy — Voice Recorder Bot

Last updated: 14 May 2026 Operator: Discord user panther13055 (ID 990361883775225926) Bot: Voice Recorder 2 (1504373717046788146)

This document explains what data the bot collects, where it's stored, how long it's kept, who it's shared with, and how you can have it deleted.

1. Plain-English summary

  • When the bot records a voice channel, your spoken audio is captured, transcribed, and saved.
  • That data is stored on the bot operator's personal computer (in recordings/<session>/) indefinitely unless deleted.
  • Recordings and transcripts are uploaded to catbox.moe / litterbox.catbox.moe to create shareable links. Anyone with a link can access the file until it expires (72 hours for audio; permanent for the polished transcript on catbox.moe unless the operator deletes it).
  • Per-server configuration data (your user ID is stored in vc_role_rules, etc.) is kept in plain-text JSON files on the operator's machine.
  • You can request deletion at any time (see §8).

If recording audio or transcripts of your voice is a problem, leave the voice channel before the bot starts recording, or ask a server administrator to add the channel to the bot's recording blacklist (,vcblacklist add #channel).

2. Data the bot collects

2.1 Voice data (captured only during an active recording session)

Data Where it comes from Format
Raw audio of every speaker Discord voice gateway 48 kHz stereo 16-bit PCM → wrapped as .wav
Speech-to-text transcript of every audible utterance Local Whisper inference on the audio above Plain text .txt

2.2 Per-server configuration (no voice data)

Stored permanently in data/guilds.json on the operator's machine. Per server:

  • Command prefix, filter words, filter-target role/channel IDs
  • VC auto-role rules — includes role IDs (does NOT include user IDs unless an admin adds one explicitly)
  • Backup channel ID
  • Command-access role ID
  • Recording-blacklist VC IDs

2.3 Global configuration

Stored permanently in data/access.json on the operator's machine:

  • Server IDs that the operator has blacklisted or whitelisted (and a reason string per blacklisted server)
  • Access-log channel ID
  • Whitelist mode (on/off)

2.4 Diagnostic logs

Stored on the operator's machine in bot.out.log and bot.err.log:

  • Bot start/stop timestamps
  • Discord connection events (no message content)
  • Errors and stack traces (may incidentally include channel or guild IDs)
  • Voice connection events (guild ID, channel ID — not audio)

2.5 Data we do not collect

  • Text messages you send in any channel (the bot does not read message content beyond what's needed to parse its own commands).
  • Direct message content unless you DM the bot directly.
  • IP addresses (Discord does not share these with bots).
  • Payment information (the bot is free).
  • Browser cookies or tracking pixels (the bot has no website).

3. Where data is stored

  • Audio + transcripts: locally on the operator's macOS machine, under recordings/<session_id>/. Not encrypted at rest. Protected by the operator's macOS user account password.
  • Configuration JSON: same machine, under data/. Same protections.
  • Shareable links: copies of the files are uploaded to:
  • litterbox.catbox.moe for audio + live transcripts (auto-deleted after 72 hours; we don't have a way to delete them earlier).
  • catbox.moe for the polished transcript (permanent unless the operator deletes it via their catbox account, which they may not have).
  • Discord channel messages: if a backup channel is configured (,backupchannel), copies of the upload links and the polished transcript file are posted into a Discord channel and stored by Discord.

The operator does not use any cloud database, analytics service, or third-party logging service beyond what is listed here.

4. Legal basis for processing (GDPR users)

For users in the EEA / UK:

  • Recording voice — processed on the basis of consent, which must be obtained by the server's organisers from every participant before recording.
  • Configuration data — processed on the basis of legitimate interest in running the bot, since the data is necessary for the requested service (e.g. you can't have per-server prefixes without storing them).
  • Diagnostic logs — processed on the basis of legitimate interest in operating and maintaining the service.

You have the rights described in §8 below.

7. Security

We take reasonable but not extensive security precautions for a free, personally-operated bot:

  • The operator's machine has full-disk encryption (FileVault) enabled.
  • Bot files are accessible only to the operator's macOS user.
  • The Discord bot token is stored in a .env file with restrictive permissions, not committed to any repository.
  • Data is not encrypted at rest beyond the operating-system-level encryption above.

We do not warrant that the data is secure against all threats. A compromise of the operator's machine, of catbox.moe, or of your Discord account could expose data. If you have higher security requirements, do not speak in voice channels where this bot is recording.

8. Your rights

Regardless of jurisdiction, you may at any time DM the operator (990361883775225926 on Discord) and request:

  • Access — a list of what data the bot has about you, including the contents of any recordings.
  • Deletion — removal of all your audio, transcripts, and any user-ID-keyed configuration. We will:
  • Delete the relevant .wav, .txt, and .pcm files from local storage.
  • Delete the permanent transcript link on catbox.moe (if any). Litterbox links will auto-expire on their own schedule and cannot be force-deleted.
  • Ask any server admin who has mirrored the recording to a backup channel to delete the mirrored message; if they refuse, that copy is outside our control.
  • Remove your user ID from any per-server configuration that contains it.
  • Correction — correction of an incorrect transcript line (we'll delete the line; we won't re-transcribe a different result).
  • Withdrawal of consent — withdrawal of consent for any future processing of your data.
  • Lodging a complaint — if you're in the EEA / UK, with your national data-protection authority.
